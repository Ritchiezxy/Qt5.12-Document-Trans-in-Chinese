详细描述

ApplicationWindow 是一个Window（窗口）类型，它可以方便地向窗口添加菜单栏、页眉 和 页脚 项。

您可以将ApplicationWindow 声明为应用程序的根项，并使用 QQmlApplicationEngine运行它。通过这种方式，您可以从QML控制窗口的属性、外观和布局。


  import QtQuick.Controls 2.12

  ApplicationWindow {
      visible: true

      menuBar: MenuBar {
          // ...
      }

      header: ToolBar {
          // ...
      }

      footer: TabBar {
          // ...
      }

      StackView {
          anchors.fill: parent
      }
  }

注意:默认情况下，ApplicationWindow是不可见的。

附加的ApplicationWindow属性

至于范围和命名解析 在QML中是如何工作的，可以在其子QML对象的任何位置引用应用程序根元素的id。尽管这种方法适用于许多应用程序和用例，但是对于通用QML组件，它可能是不可接受的，因为它创建了对周围环境的依赖。

ApplicationWindow 提供了一组附加属性，可用于从无法直接访问窗口的位置访问窗口及其构建基块，而无需创建对某个窗口id的依赖关系。 使用ApplicationWindow附加属性的QML组件在任何窗口中工作，而不管其id如何。

请参见 定制ApplicationWindow、Overlay、 Page、 容器控件 和 Qt Quick Controls 2中的焦点管理。

属性文档

[read-only] activeFocusControl : Control

此属性保存当前具有活动焦点的控件，如果没有具有活动焦点的控件，则为null。

Window::activeFocusItem 和 ApplicationWindow::activeFocusControl的区别在于前者可能指向控件的构建块，而后者指向封闭控件。例如，当SpinBox 焦点时，activeFocusItem指向编辑器，activeFocusControl 指向 SpinBox 本身。

请参见 Window::activeFocusItem.


background : Item

此属性保存背景项。

背景项堆叠在内容项之下，但在窗口的背景颜色之上。

例如，背景项对于图像和渐变是有用的，但是color 属性对于纯色更可取，因为它不需要创建项。

注意:如果背景项没有指定显式大小，它将自动跟随控件的大小。在大多数情况下，不需要为背景项指定宽度或高度。

请参见 定制ApplicationWindow、contentItem、header（页眉） 和 footer（页脚）。


[default] contentData : list<Object>

此默认属性保存作为窗口子元素声明的所有对象的列表。

data属性允许您在ApplicationWindow中自由地混合可视化子窗口、资源和其他窗口。

如果将一个项分配给contentData列表，它将成为窗口的contentItem的一个子元素，因此它将出现在窗口中。该项的父项将是窗口的contentItem。

通常不需要引用contentData属性，因为它是ApplicationWindow的默认属性，因此所有子项目都自动分配给该属性。

请参见 contentItem。


[read-only] contentItem : Item

此属性保存窗口内容项。

内容项堆叠在background（背景）项之上， 并在menuBar（菜单栏）、header（页眉）和 footer（页脚）项之下。

请参见 background, menuBar, header 和 footer。


font : font

此属性保存窗口当前设置的字体。

默认字体取决于系统环境。QGuiApplication 维护一个system/theme字体，作为所有应用程序窗口的默认字体。在加载任何QML之前，还可以通过将自定义字体传递给QGuiApplication::setFont()来设置窗口的默认字体。最后，将字体与Qt的字体数据库进行匹配，以找到最佳匹配。

ApplicationWindow 将显式字体属性传播到子控件。如果更改窗口字体上的特定属性，该属性将传播到窗口中的所有子控件，覆盖该属性的任何系统默认值。

请参见 Control::font。


footer : Item

此属性保存窗口页脚项。页脚项定位到底部，并调整为窗口的宽度。默认值为null。


  ApplicationWindow {
      footer: ToolBar {
          // ...
      }
  }

注意:将ToolBar（工具栏）、 TabBar（选项卡）、或DialogButtonBox（对话框按钮框） 指定为窗口页脚会自动将相应的ToolBar::position、 TabBar::position或DialogButtonBox::position属性设置为Footer（页脚）。

请参见 menuBar, header, and Page::footer.


header : Item

此属性保存窗口标题项。标题项位于窗口顶部，菜单栏下方，并调整为窗口的宽度。默认值为null。


  ApplicationWindow {
      header: TabBar {
          // ...
      }
  }

注意：将ToolBar（工具栏）、TabBar（选项卡）或DialogButtonBox（对话框按钮框） 指定为窗口页眉会自动将相应的 ToolBar::position、 TabBar::position或DialogButtonBox::position 属性设置为 Header（页眉）。

请参见 menuBar, footer 和 Page::header。


locale : Locale

此属性保存窗口的区域设置。

默认区域设置取决于系统环境。在加载任何QML之前，可以通过调用QLocale::setDefault()来设置默认区域设置。

ApplicationWindow 将区域设置传播到子控件。如果更改窗口的区域设置，该区域设置将传播到窗口中的所有子控件，覆盖系统默认区域设置。

请参见 Control::locale 和 LayoutMirroring。


menuBar : Item

This property holds the window menu bar. The menu bar is positioned at the top of the window, above the header, and resized to the width of the window. The default value is null.


  ApplicationWindow {
      menuBar: MenuBar {
          // ...
      }
  }

This property was introduced in QtQuick.Controls 2.3 (Qt 5.10).

请参见 header, footer, and MenuBar.


palette : palette

This property holds the palette currently set for the window.

The default palette depends on the system environment. QGuiApplication maintains a system/theme palette which serves as a default for all application windows. You can also set the default palette for windows by passing a custom palette to QGuiApplication::setPalette(), before loading any QML.

ApplicationWindow propagates explicit palette properties to child controls. If you change a specific property on the window's palette, that property propagates to all child controls in the window, overriding any system defaults for that property.

This property was introduced in QtQuick.Controls 2.3 (Qt 5.10).

请参见 Control::palette, Popup::palette, and palette QML Basic Type.


Attached Property Documentation

[read-only] ApplicationWindow.activeFocusControl : Control

This attached property holds the control that currently has active focus, or null if there is no control with active focus. The property can be attached to any item. The value is null if the item is not in a window, or the window has no active focus.

请参见 Window::activeFocusItem and 附加的ApplicationWindow属性.


[read-only] ApplicationWindow.contentItem : Item

This attached property holds the window content item. The property can be attached to any item. The value is null if the item is not in an ApplicationWindow.

请参见 附加的ApplicationWindow属性.


[read-only] ApplicationWindow.footer : Item

This attached property holds the window footer item. The property can be attached to any item. The value is null if the item is not in an ApplicationWindow, or the window has no footer item.

请参见 附加的ApplicationWindow属性.


[read-only] ApplicationWindow.header : Item

This attached property holds the window header item. The property can be attached to any item. The value is null if the item is not in an ApplicationWindow, or the window has no header item.

请参见 附加的ApplicationWindow属性.


[read-only] ApplicationWindow.menuBar : Item

This attached property holds the window menu bar. The property can be attached to any item. The value is null if the item is not in an ApplicationWindow, or the window has no menu bar.

This property was introduced in QtQuick.Controls 2.3 (Qt 5.10).

请参见 附加的ApplicationWindow属性.


[read-only] ApplicationWindow.window : ApplicationWindow

This attached property holds the application window. The property can be attached to any item. The value is null if the item is not in an ApplicationWindow.

请参见 附加的ApplicationWindow属性.
