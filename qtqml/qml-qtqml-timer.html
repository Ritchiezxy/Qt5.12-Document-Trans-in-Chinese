详细描述

定时器可以用来触发一个动作，一次，或重复在一个给定的间隔。

这里有一个计时器，它显示当前日期和时间，并每500毫秒更新一次文本。它使用JavaScript Date 对象来访问当前时间。


  import QtQuick 2.0

  Item {
      Timer {
          interval: 500; running: true; repeat: true
          onTriggered: time.text = Date().toString()
      }

      Text { id: time }
  }

定时器类型与动画定时器同步。由于动画定时器通常设置为60fps，所以定时器的分辨率最多为16ms。

如果计时器正在运行，并且其中一个属性被更改，则运行时间将重置。例如，如果间隔为1000ms的计时器在启动后将其repeat属性更改为500ms，则运行时间将重置为0，并在1000ms之后触发计时器。

参见 Qt Quick演示-时钟。

属性文档

interval : int

设置触发器之间的interval（间隔），以毫秒为单位。

默认间隔是1000毫秒。


repeat : bool

如果repeat 为真，则在指定的时间间隔内重复触发计时器;否则，定时器将在指定的时间间隔触发一次，然后停止(即运行将设置为false)。

repeat 默认值为false。

参见 running。


running : bool

如果设置为true，则启动计时器;否则将停止计时器。对于非重复计时器，在触发计时器后将running设置为false。

running 默认值为false。

参见repeat。


triggeredOnStart : bool

当计时器启动时，第一个触发器通常在指定的间隔之后。有时需要在计时器启动时立即触发;例如，建立初始状态。

如果triggeredOnStart为真，则在启动时立即触发计时器，然后在指定的时间间隔触发计时器。注意，如果repeat设置为false，则触发计时器两次;一次开始，一次间隔。

triggeredOnStart 默认为false。

参见running。


信号文档

triggered()

此信号在计时器超时时发出。

对应的处理程序是onTriggered。


方法文档

restart()

重新启动定时器

如果计时器没有运行，它将被启动，否则将被停止，重置为初始状态并启动。在调用restart()之后，running 属性将为true。


start()

启动定时器

如果计时器已经在运行，则调用此方法没有效果。调用start()后，running属性将为true。


stop()

停止计时器

如果计时器不运行，则调用此方法无效。调用stop()后，running的属性将为false。
